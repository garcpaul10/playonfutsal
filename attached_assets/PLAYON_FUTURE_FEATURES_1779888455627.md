## How an idea moves through this doc

1. **Inbox** — raw idea, captured fast, no judgment.
2. **Triage** — sort into a category below; add a one-line "why" and rough effort.
3. **Promote** — when ready, copy the detail into the OVERVIEW + PROMPT, bump their versions, and mark the item ✅ here with the version it landed in.
4. **Decline** — if it's decided against, mark ❌ with a short reason so it doesn't keep resurfacing.

## Already-known "later" items (parked from planning)

These came up during planning and were intentionally deferred. They live here until promoted.

| Idea | Status | Notes |
|---|---|---|
| **Memberships / subscriptions** | 🔍 Evaluating | Structure exists in the plan (Phase 13), but tiers/benefits/billing cadence undecided. Promote once you define what a membership gets you. |
| **Native mobile app** | 💡 Idea | Phase 15 in the build plan. Web-first; native after web is stable. |
| **Multi-location / franchise** | 💡 Idea | Data model is built not to block it; not built now (single location — Alumni Center). |
| **Open public API for third parties** | 💡 Idea | Internal API is being built API-first; a documented public API is a later growth item. |

| **SMS provider selection** | 🔍 Evaluating | SMS/text is in scope; pick the provider (e.g., Twilio) at build time. |
